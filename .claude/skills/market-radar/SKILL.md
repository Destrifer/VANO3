# Market Radar — рыночный радар printmos.ru

Один прогон = дельта по конкурентам + обновлённый бэклог идей. Скилл **только
предлагает**: ничего не публикует в Directus, не правит контент, не коммитит.
Каждое предложение — со ссылкой на источник (URL конкурента / цифра спроса).

## Обычный прогон (раз в 1–2 недели, свежие XLS НЕ нужны)

1. **Контекст:** прочитать `seo/market-radar.md` (шапка: № прогона, очередь ротации,
   открытые идеи) и `seo/radar/competitors.json`.

2. **Наш срез:** `python .claude/skills/market-radar/scripts/our_catalog.py` —
   хабы/кластеры с прода (Directus, токен из `.env`). Сравнить с прошлым прогоном:
   что мы закрыли — пометить идеи в бэклоге `done`.

3. **Sitemap-дельта:** `python .claude/skills/market-radar/scripts/fetch_sitemaps.py`
   (фоном — работает минут 10: вежливые паузы 2–5 с, браузерный UA, ретраи; так
   хостинги не банят). Печатает дельту к прошлому снапшоту. Новые URL конкурентов =
   их свежие гипотезы о спросе — главный сигнал прогона.
   При необходимости тематической картины: `python .../classify_urls.py <hosts...>`
   (матрица тема×хост по последнему снапшоту; темы с ★ = наши пробелы, ⛔ = EXCLUDE).

4. **Ротация-глубина (2 задачи за прогон, очередь — в шапке бэклога):**
   - *Функционал хоста:* 1 хост костяка разобрать живьём в браузере (страницы
     JS-driven — только браузер, не curl): калькулятор, пути заказа, блоки страницы.
     Сверять с `seo/page-anatomy-benchmark.md`, новое — дописывать туда.
   - *SERP-проба:* 2–3 наших раздела, по 2–3 коммерческих запроса (WebSearch:
     «печать X москва»). Цель — новые участники рынка. Новый хост в топе →
     квалификация: sitemap (доля полиграфии, число посадочных, свежесть lastmod) +
     главная (типография? Москва/федеральный онлайн? калькулятор?) → предложить
     пользователю корзину (tier_a/watchlist/reject). Reject — всегда с причиной,
     чтобы не пережёвывать.

5. **Свежие XLS (опционально):** проверить `C:\Users\user\Downloads\` на
   `wordcraft-tables-info-*.xlsx` новее прошлого прогона → скопировать в
   `webmaster-raw`. Если накопилось ≥10 новых или прогон 3-й по счёту — ревизия
   костяка (см. ниже).

6. **Обновить `seo/market-radar.md`:**
   - шапка: дата, № прогона, следующая пара ротации, рекомендуемая дата следующего
     прогона (+2 недели);
   - секция «Дельта прогона #N»: что нового у конкурентов / что закрыли мы /
     новички рынка;
   - таблицы «вширь»/«вглубь»: новые идеи со статусом `new`, источником и
     приоритетом 🔴🟡🟢; закрытые — `done`, отвергнутые — `rejected` (не удалять).

7. **Сводка пользователю:** топ-3 «вглубь», топ-3 «вширь», функциональные находки,
   новички. Кратко, с ссылками на файлы.

## Ревизия костяка (каждый ~3-й прогон или по команде)

Пересчитать скоринг: `python .claude/skills/market-radar/scripts/radar_hosts.py`
(читает ВСЕ xlsx из `webmaster-raw`; путь можно переопределить env `RADAR_RAW`).
Скоринг: единица счёта — раздел (не запрос — спам вариациями не работает);
сила присутствия = 0.6×ранг в HostsTable + 0.4×число посадочных в UrlsTable;
вес раздела = log10(спрос). Автостоп-лист агрегаторов — в скрипте (STOP).
Изменения состава tier_a/watchlist предложить пользователю, применить в
`competitors.json` только после его подтверждения, с датой.

Если для новых разделов нет выгрузок — попросить пользователя выгрузить сиды из
Вебмастера («Подбор запросов и анализ рынка», регион Москва, «Скачать таблицы →
XLS» в Downloads) и перечислить, какие именно.

## Массовые разборы — дешёвыми воркерами

Разбор сотен слагов/страниц (группировка вариантов тем, сравнение с нашими
кластерами) — раскидывать на 2–3 haiku-субагентов (Agent, model haiku), каждому:
свой список тем/файлов, наши кластеры для сравнения (из `our_catalog.json`),
формат ответа. Синтез и приоритеты — самому.

**Baseline в промте воркера = материалы+опции со страницы И полный список наших
кластеров хаба И смежные хабы** (`our_catalog.json`). Воркер знает только то, что
в промте: чего там нет — он объявит гэпом. Проверено трижды (визитки, брошюры,
наклейки) — каждый раз половина «находок» оказывалась тем, что у нас уже есть,
вплоть до целого хаба `/volume-stickers`. **Все `[НОВОЕ]` от воркера сверять с
`our_catalog.json` руками перед записью в отчёт** — воркеру на слово не верить.
Полный метод гэп-аудита хаба — в `seo/market-radar.md`, блок «Пообъектный
гэп-аудит хаба (метод)».

## Правила

- Наши EXCLUDE (не предлагать): сувенирка/мерч, широкоформат, печати-штампы,
  DTF/ткань, наклейки на авто, голография, ПВХ/металл-таблички, фотоподарки.
  Сверяться с `seo/roadmap-sections.md` («сознательно НЕ делаем»).
- Известные ловушки данных: printkov и digital-printing — /catalog = мерч;
  copy.ru — региональные копии (срезаются в classify_urls); kopirka — /city/,
  шаблоны с числовыми суффиксами; multifoto — город первым сегментом.
- Снапшоты и промежуточные json — ЛОКАЛЬНЫЕ (в `seo/.gitignore`, ~19 МБ/прогон);
  в гит идут только конфиг и бэклог. Помнить: push в master = автодеплой —
  коммитить только по просьбе пользователя.
