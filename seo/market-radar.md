# Рыночный радар printmos.ru — живой бэклог

**Процесс:** скилл `/market-radar` (`.claude/skills/market-radar/SKILL.md`), конфиг конкурентов — `seo/radar/competitors.json`.

> ℹ️ **Как вносить правки контента в прод** (проверено 2026-07-16 на правке «фальц»):
> - **Токен для записи — `DIRECTUS_ADMIN_TOKEN` в `.env`.** Не путать с `DIRECTUS_TOKEN` (он локальный, на проде даёт `INVALID_CREDENTIALS` 401). База прода — `https://admin.printmos.ru`, а `DIRECTUS_URL`/`DIRECTUS_PUBLIC_URL` в `.env` смотрят на `localhost:8055`, т.е. хост подставлять руками. Чтение прода открыто анонимно, запись — только с токеном.
> - **Контент не в гите.** Тексты страниц — `products.intro_text`, `promoted_pages` (h1/intro_text/preset/tile_label), `faq_items`. Кластеры лежат в `promoted_pages`, НЕ в `products`.
> - ⚠️ **Только utf-8** (`json.dumps(..., ensure_ascii=False).encode('utf-8')`, заголовок `charset=utf-8`) — иначе текст можно угробить безвозвратно, прецедент был (`TECHDEBT.md` §2, papers 17–26).
> - ⚠️ **HTML — только в `intro_text`.** `Faq.astro` рендерит ответы с экранированием: теги вылезут в текст и в JSON-LD.
> - **Пересборка запускается сама:** Directus Flow «Deploy site on publish» (`items.create/update/delete` по products/promoted_pages/faq_items/…) POST-ит `event_type: deploy-site` в GitHub dispatches. Несколько правок подряд схлопываются в одну сборку (`concurrency: cancel-in-progress`). Цикл — ~4–5 мин до появления на сайте; сайт SSG, до пересборки правка не видна.
> - **Пуш доков сборку НЕ запускает:** в `.github/workflows/deploy.yml` стоит `paths-ignore` на `**.md`, `docs/**`, `seo/**`, `ops/**`. Для ручного запуска — `workflow_dispatch` (в этом окружении нет `gh` CLI).

## Пообъектный гэп-аудит хаба (метод)

Отдельный от общего прогона режим: берём ОДИН наш хаб и сверяем его начинку со страницами-аналогами всех конкурентов костяка.

**Процесс:**
1. Снять baseline нашего хаба — из **трёх** источников, иначе воркеры выдумают гэпы:
   - `WebFetch` нашей страницы (`printmos.ru/<slug>/`) → материалы + опции калькулятора + функционал;
   - **наши кластеры хаба** из `seo/radar/our_catalog.json` (полным списком, в промт воркера);
   - **смежные хабы**: прогнать `our_catalog.json` на родственные слаги (напр. `stickers` → `labels`, `volume-stickers`) и перечислить их воркеру как «у нас есть отдельно».
2. Разбить костяк (12 хостов) на 3 батча по 4, запустить 3 дешёвых воркера (`Agent`, model=haiku, general-purpose, WebFetch/WebSearch). Каждому передать полный baseline (все три источника) и просить помечать `[НОВОЕ]` то, чего у нас нет. ~2 мин, ~40k токенов/батч.
3. **Отфильтровать ложные гэпы вручную** — воркер видит только baseline из промта, наших кластеров и смежных хабов он не знает. Каждый `[НОВОЕ]` сверить с baseline и `our_catalog.json` перед тем, как писать в отчёт.
4. Свести в таблицу с колонкой «у скольких из 12», сгруппировав: **материалы / опции / функционал / посадочные**. Сортировать по частоте — частое = приоритет.
5. Перед постройкой кластеров — сверить с ядром (урок каталогов: спрос может отсутствовать даже там, где у конкурентов сетка).

> ⚠️ **Ложные гэпы — системная болезнь режима.** На всех трёх аудитах воркеры «находили» то, что у нас давно есть: визитки — конгрев, скругление углов, плоский УФ-лак; брошюры — soft-touch, голографич. фольга; наклейки — Soft Touch, голографич. фольга, водостойкие, виниловые, круглые, фигурные, светоотражающие, плоттерная резка, **и целый наш хаб `/volume-stickers` (эпоксидная смола)**, который все 3 воркера записали в «нет у нас». Причина всегда одна: baseline в промте беднее нашего реального ассортимента. Отсюда п.1 и п.3. Побочный сигнал: если воркер с веб-поиском не нашёл наш продукт — его, скорее всего, не находят и пользователи (→ перелинковка/SEO).

> ⚠️ **Здесь ядро слепо — и это важный урок метода.** «Поштучно/1 экземпляр» = 0 запросов **не означает отсутствия спроса**: так не формулируют. Покупатель ищет `свадебная фотокнига` (556) и по умолчанию имеет в виду один экземпляр себе — ожидание зашито в продукт, а не в запрос. Там, где ожидание подразумевается продуктом (тираж, форма выпуска, базовое свойство), нули ядра — артефакт, и опираться надо на конкурентов. Ср. этикетки: там «рулонные» **искали явно** (11.4к), потому что это осознанный выбор формы выпуска, — и ядро сработало.

**Выпускные 4.1к** внутри ядра фотокниг → это наш отдельный хаб `/graduation-albums` (71.9к своим ядром), не гэп.
