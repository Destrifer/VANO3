## Процесс на каждый раздел (повторяемый цикл)
1. **Владелец:** экспорт ядра из Я.Вебмастера (регион Москва) → `van2/seo/SEO-ядро_<раздел>.xlsx`.
2. **Claude:** Tier-разметка + чистка мусора (Шаг 4.5 PLAYBOOK) → `..._Tier.xlsx`.
3. Архитектура: хаб + кластеры (Tier-1 плитки / Tier-2 без плитки) + вхождения (Tier-3).
4. **Конфиг продукта:** часто пустой/не того типа — донастроить (размеры/материалы/finishing; `multipage` для книг/журналов; см. как делали catalogs/menus/labels).
5. Хаб + кластеры + рыбка-плейсхолдер → сборка/превью (проверять пресет ПОСЛЕ гидрации).
6. Цены/факты — черновик по конкурентам, пометка на выверку.

> Малые STANDALONE без выраженных модификаторов можно собрать **из caps без ядра** (один хаб, без кластеров) — но ядро всё равно полезно для вхождений/чистки.

- **Этикетки:** рулонные/ленточные, флексопечать, термотрансфер, **честный знак/переменные данные** — EXCLUDE.
- Авто-наклейки, DTF/УФ-перенос, голографические, магнитные наклейки — EXCLUDE.
