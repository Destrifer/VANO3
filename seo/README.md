# SEO · сбор ядра и анализ конкурентов

➡️ **Полная стратегия и повторяемый процесс — в `PLAYBOOK.md` (входная точка).** Этот файл — только детали пайплайна данных.

Рабочая папка для семантики и конкурентного анализа. Источник данных — экспорты XLS из Яндекс.Вебмастера («Подбор запросов и анализ рынка», регион Москва).

## Процесс (разделение труда)

1. **Ты** в Вебмастере выставляешь регион = Москва, прогоняешь сид, жмёшь «Скачать таблицы → XLS», кладёшь файл в `C:\Users\user\Downloads\` (или сюда в `queries/`).
2. **Я** запускаю пайплайн: парсю xlsx → чищу → кластеризую → свожу в дерево и в конкурентов.
3. Повторяем по веткам (drill-down): сначала верхний уровень (`визитки`), потом каждую ветку отдельным сидом (`печать визиток`, `визитки тиснение`, …).

## Пайплайн (скрипты, переиспользуемые)

- `_xlsx_dump.py <file.xlsx> <out.json>` — парсит xlsx (без внешних либ) → JSON всех вкладок.
- `_analyze.py` — читает `dump.json` → классифицирует (PRINT / INFO / BRANDED / NONTARGET), агрегирует по кластерам, считает конкурентов → пишет отчёты и CSV.

Запуск:
```
python _xlsx_dump.py "<путь к xlsx>" dump.json
python _analyze.py
```

## Схема данных (`data/`)

Накопительно, по одному набору файлов на сид (суффикс = сид):

- `queries_<сид>.csv` — `query, cluster, demand, clicks, competitiveness, category`
- `competitors_<сид>.csv` — `host, pages_count, queries_sum`
- `competitor_urls_<сид>.csv` — `url, title, queries, host`

## Артефакты-итоги

- `keyword-tree.md` — **дерево интентов** (ветка → под-кластеры → решение landing/intext/guide/reject), главный рабочий документ.
- `report_clusters.txt`, `report_competitors.txt` — сырые читаемые отчёты последнего прогона.

## Категории классификатора

- **PRINT** — наш коммерческий интент (печать/изготовление + модификаторы).
- **INFO** — информационные (гайды), не коммерческие посадочные.
- **BRANDED** — запросы с брендами конкурентов (навигационные); не наши цели, но сигнал рынка.
- **NONTARGET** — чужой бизнес: сайт-визитка (веб-разработка), онлайн-генераторы, ИИ/нейросети, электронная/видео-визитка, продвижение сайтов.

Стоп-листы и бренд-детект — в `_analyze.py` (`NONTARGET`, `INFO`, `LATIN_OK`).
