# 06 · Доставка и оплата

Трек спроектирован, **код пока не написан**. Оплата онлайн — место в UI зарезервировано
(«скоро»). Сейчас заказ = заявка (см. doc 01, Маршрут 1; П2 — цена из конфигуратора = цена в счёте).

## Принципы

- **Провайдеро-независимая модель (адаптеры) + конфиг данными в Directus** — как стратегии
  цены (doc 03), макеты превью (doc 02), конвейер скидок. Добавление провайдера ≠ переписывание.
- **Без договоров на старте.** Предлагаем много способов доставки как «выбор + захват
  адреса/ПВЗ», физическую отгрузку делает менеджер в кабинете провайдера. Автоматизация
  (тариф / создание отправления / трекинг) подключается **по одному провайдеру позже**.
- **Сервер — источник истины** (П2): стоимость доставки и статус оплаты считаются/проверяются
  на сервере, клиентским значениям не верим (как и цене товара).

## Оплата

Методы (выбраны владельцем):
1. **Онлайн-предоплата** (карты/СБП) — место в UI зарезервировано, помечено «скоро»
   (кнопка disabled). Реализуется в отдельной фазе.
2. **При получении** — наличные/карта на самовывозе или курьеру (постоплата). Без интеграции.
3. **Счёт для юрлиц** — безнал по счёту (реквизиты), постоплата переводом, ведёт менеджер.

Поля заказа: `payment_method` (online / on_receipt / invoice), `payment_status`
(unpaid / pending / paid / refunded), `payment_id` (id платежа шлюза).

Онлайн-оплата (позже): шлюз **абстрактно** (ЮKassa / Тинькофф — решим при реализации).
Поток: создать платёж в шлюзе (секрет в `.env`) → редирект на оплату →
**webhook** `/api/payment/webhook` (проверка подписи, идемпотентно) → `payment_status=paid`.
Статусу «оплачено» с клиента **не верим** — только из webhook.

B2B-счёт (позже): генерация PDF-счёта с реквизитами, постоплата переводом, подтверждает менеджер.

## Доставка

### Модель

Каждый способ — запись-провайдер (Directus, напр. `delivery_methods`):
- `name`, `type` (`pickup` / `courier` / `pvz` / `postamat`),
- `integration` (`manual` / `widget` / `api`),
- `cost_rule` (`free` / `fixed` / `zone` / `manual` / `api`), `cost`, `zone`, `settings` (JSON),
- `active`, `sort`.

### Режимы интеграции (ключевое для «без договора»)

- **`manual`** — выбор способа + адрес/комментарий; цена `free`/`fixed`/«уточнит менеджер»;
  отгрузка вручную. **Договор не нужен.**
- **`widget`** — встраиваемый виджет выбора ПВЗ (карты СДЭК / Яндекс Маркет / Boxberry);
  захватываем код+адрес точки. **Отображение/выбор обычно без договора**; отгрузка — менеджер
  (manual) или API позже.
- **`api`** — тариф + создание отправления + трекинг через адаптер. **Требует
  регистрации/договора** (у части есть sandbox: СДЭК тест, Borzo/Dostavista песочница).

### Что доступно без договора (на старте)

- ПВЗ-виджеты (карта точек) — встраиваются бесплатно, выбор точки без договора → сохраняем `pvz_code`/`pvz_address`.
- Тариф/отправление/трекинг — нужна регистрация/договор; проверять **по каждому провайдеру**
  (условия меняются).
- **Старт = `manual` + `widget`** (много способов, ноль договоров); `api`-адаптеры — позже.

### Целевые провайдеры

СДЭК, Boxberry, 5Post, Ozon (ПВЗ), Яндекс Маркет (ПВЗ), Яндекс Go (курьер-такси),
Dostavista/Borzo (курьер), постаматы, самовывоз. **Не включаем** формат «мастодонтов»
(Почта России, Деловые Линии) — иной формат и документооборот.

### Стоимость

Считается **на сервере**: `free`=0, `fixed`=ставка из настроек, `api`=из адаптера,
«по запросу»=0/уточнит менеджер. Итог: `goods_total − discount + delivery_cost = total`.

## Шаг оформления (checkout, UI)

Расширить форму корзины:
- **Доставка** — выбор провайдера; курьер → адрес; ПВЗ → виджет/выбор точки; показать

- **Оплата** — онлайн (disabled, «скоро») / при получении / счёт юрлицу.
- Итог пересчитывается с доставкой (клиент — оценка, сервер — авторитетно при создании заказа).

## Архитектура (адаптер)

```
interface DeliveryProvider {
  quote(dest, order): number | null;     // стоимость (или null = уточнит менеджер)
  createShipment?(order): Promise<...>;   // позже, при наличии API/договора
  track?(id): Promise<...>;               // позже
}
```
Провайдеры = данные в Directus + код-адаптеры по `integration`. MVP: `ManualProvider`
(cost fixed/null). Позже: `CdekProvider`, `YandexProvider`, `BorzoProvider`… — за одним интерфейсом.

## Фазы

1. **Поля заказа + checkout-шаг**: выбор доставки/оплаты, адрес/ПВЗ-виджет, доставка в сумму.
   Оплата: при получении / счёт + онлайн-заглушка «скоро». **Без внешних API.**
2. **ПВЗ-виджеты** (СДЭК/Яндекс Маркет) для выбора точки — без договора.
3. **Онлайн-оплата** (шлюз) + webhook + `payment_status`.
4. **API-адаптеры доставки** per-provider (по мере договоров): тариф/отправление/трекинг.
5. **Счёт для юрлиц** (PDF).
