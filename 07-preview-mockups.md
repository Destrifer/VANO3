## Суть

В калькуляторе есть canvas-предпросмотр («Предпросмотр материала и формы»). Он
показывает, **как выглядит изделие**: форма, размер, бумага, ламинация, фольга.

Изначально сцен было мало (4 общих: визитка/наклейка/листовка/бланк) — почти все
продукты падали на дефолтную визитку, и баннер или бирка показывали «PM / Иван
Петров». Мы перешли на **пообъектные куклы**: у каждого продукта свой образ,
спроектированный от его кластеров.

**Зачем это, кроме красоты:** превью попадает не только на страницу продукта, но и
в корзину (снимок канваса), а в перспективе — в плитки каталога. Один образ
работает в трёх местах.

## Инварианты (главное, не нарушать)

### 1. Сцена рисует ТОЛЬКО «куклу», материалы — движок

`Preview.vue` даёт универсальные слои: контур/форма/скругление, тень, текстура
бумаги, глянец ламинации, металлик, фольга. Сцена в `mockups.ts` рисует **только
ink-контент** (`content()`), фольгу — через общий `drawFoilText()` в `foil()`.

**Почему:** чтобы одна и та же ламинация или фольга выглядела **одинаково на всех
продуктах**. Иначе 25 сцен = 25 разных хардкодов и зоопарк эффектов.

> **Прецедент-исключение:** если эффект **и есть сам продукт**, он рисуется в
> сцене. Пример — `volume-sticker` (объёмная наклейка со смолой): эпоксидный купол
> это форма изделия, а не постобработка. Ламинация даёт плоский блик, купол —
> объём. Критерий: «это отделка поверх любого продукта» → движок; «без этого
> продукта не существует» → сцена.

### 2. Кластеры определяют вид куклы

Смотрим кластеры продукта (`promoted_pages`) и делаем **один** образ, который их
покрывает. Различия кластеров — это обычно **материал и форма**, а их даёт движок.

*Эталонный пример — бланки (3 кластера):* рисуем БСО-бланк (серия/№ — признак
строгой отчётности, покрывает «БСО» и «самокопирующиеся», их часто и печатают на
самокопирке) **+ условное лого** (покрывает «фирменные»). Один рисунок = 3 кластера.

*Второй пример — визитки (10 кластеров):* VIP/ламинация/УФ/фольга/металл/пластик/
круглые — это всё материал и форма. Значит кукла одна, качественная, а вариативность
приезжает из движка.

### 3. Метрики — от МЕНЬШЕЙ стороны

Шрифты и отступы считать от `Math.min(w, h)`, не от высоты и не от ширины.

**Грабли (реальные):** сцена `ticket` была свёрстана под ландшафт со шрифтами от
высоты, а дефолтный размер билета — Евро 99×210 (**портрет**) → текст раздулся и
вылез за края. Если у продукта возможны обе ориентации — ветвить раскладку
(см. `ticket`: портрет = перфорация снизу, ландшафт = перфорация справа).

Также следить, чтобы текст не заходил на кайму/край (ловили в `business-card`).

### 4. У multipage свой stage и свой реестр — но тот же ключ

Многостраничные (книги, журналы, брошюры, фотокниги, ежедневники, каталоги,
тетради, блокноты, выпускные альбомы — 9 шт) рисует `BookletPreview.vue`, и
сцену он берёт из **`covers.ts`**, а не из `mockups.ts`. Ключ общий —
`products.preview_kind`, просто резолвится в своём реестре; фолбэк — `booklet`.

**Почему два реестра:** у обложки другое окружение (переплёт, полосы,
разлиновка) и другие слои вокруг (толщина блока, кант, кольца, ремешок).
Один реестр означал бы, что половина сцен получает бессмысленный `env`.

> До 2026-07-23 multipage `preview_kind` не читал вовсе, и **все 39 кластеров
> девяти продуктов показывали одну обложку** «Брошюра · каталог 2026»,
> захардкоженную в компоненте.

### 4a. Конструкцию рисует движок, объявляет — сцена

Твёрдый переплёт, кольца, ремешок с ляссе, вырубное окно — это не ink, но и не
выбор пользователя. Сцена объявляет их флагом (`features: { rings, strap }`),
рисует движок — чтобы кольца тетради и кольца любого будущего продукта
выглядели одинаково. Вырубное окно — общий примитив `dieCutWindow()`.

Исключение то же, что у `volume-sticker`: если изделие физически **не книжка**,
сцена может отрисовать себя целиком через `render()`. Пока такая одна — газета.

### 5. Пустой `preview_kind` молча падает на `card`

Поэтому «продукт выглядит как визитка» = сцена не назначена либо назначено имя,
которого нет в реестре.

## Архитектура

| Файл | Роль |
|---|---|
| `src/lib/preview/primitives.ts` | чистые ctx-хелперы: `shapePath`, `paperTexture`, `laminationGloss`, `drawFoilText`, `inkColor`, `roundRect` |
| `src/lib/preview/mockups.ts` | **реестр листовых сцен**: `content()` + опц. `foil()`; `getMockup(kind)` с фолбэком на `card` |
| `src/lib/preview/covers.ts` | **реестр обложек** multipage + `bindingKindOf()`; `getCover(kind)` с фолбэком на `booklet` |
| `src/components/calculator/Preview.vue` | stage листового: контур → материя → сцена → глянец → фольга |
| `src/components/calculator/BookletPreview.vue` | stage книжки: корешок/блок → обложка → материя → сцена → глянец → фольга → переплёт → аксессуары |

Контракт сцены:

```ts
type Mockup = {
  content: (ctx, r: Rect, env: MockupEnv) => void;  // ink-«кукла»
  foil?:   (ctx, r: Rect, env: MockupEnv) => void;  // металл поверх глянца
};
// env = { round, ink, foilOn, foilHex, foldCount }
```

Контракт обложки (`covers.ts`):

```ts
type Cover = {
  content: (ctx, r: Rect, env: CoverEnv) => void;  // ink на обложке
  foil?:   (ctx, r: Rect, env: CoverEnv) => void;
  features?: { rings?: boolean; strap?: boolean };  // рисует движок
  render?: (ctx, cssW, cssH, env) => void;          // self-render (газета)
};
// env = { ink, cover, foilOn, foilHex, binding, pages, ruling, mm }
// binding: 'staple' | 'spiral' | 'glue' | 'hardcover'
```

## Как назначить сцену продукту

Сцена выбирается полем Directus **`products.preview_kind`** (значение = ключ в
реестре `mockups`).

- Поле правится **на ПРОДЕ** (admin.printmos.ru), токен `DIRECTUS_ADMIN_TOKEN`
  из `.env` (не `DIRECTUS_TOKEN` — тот 401).
- У поля есть условие видимости в админке: `strategy _nin [sheet, fixed] → hidden`
  (расширено с «только sheet», иначе конверты/папки/3D-наклейки не настраивались).
  **Теперь его нужно расширить и на `multipage`** — обложки читают тот же ключ,
  а поле для книжек в админке пока скрыто. Это правка схемы → снапшот
  обязателен (см. ниже). Через API значение пишется и без этого.
- Правка **значения** схему не меняет → снапшот не нужен. Правка **условия/типа
  поля** = правка схемы → обязателен `ops/schema-snapshot.sh` + коммит, иначе
  следующий деплой откатит.

### Порядок выкатки (важен!)

1. **Сначала код** сцены → `git push` в `master` → дождаться успеха сборки.
2. **Потом** `preview_kind` в Directus. Если наоборот — будет окно, где значение
   уже стоит, а сцены в бандле ещё нет → молчаливый фолбэк на визитку.
3. Правка `preview_kind` дёргает Flow → `repository_dispatch: deploy-site` →
   пересборка статики. **Задержка ~8–9 минут**, это нормально.

**Экономия:** делать пачкой — все сцены одним пушем, все `preview_kind` одним
батчем, одна пересборка. Иначе 20+ циклов по 9 минут.

## Где ещё видно превью

- **Корзина** — не иконка и не живой канвас: `Preview.vue::captureThumb()` снимает
  JPEG-dataURL 240px, `OrderPlate.vue` кладёт его в строку корзины (`cart.ts`),
  `CartView.vue` рисует `<img>`. Это **замороженный снимок с выбором пользователя**.
  Следствие: что нарисовали в сцене — корзина получает бесплатно.
- **Каталог `/catalog`** — общая страница всех продуктов по категориям. Визуал
  плитки живёт в `.product-tile__media` (сейчас там иконка `astro-icon`).
  Кандидат на замену куклой — см. «Что дальше».

## Рецепт: добавить новую сцену

1. Посмотреть кластеры продукта:
   `GET /items/promoted_pages?filter[product][slug][_eq]=<slug>&fields=slug,tile_label,h1`
2. Придумать один образ, покрывающий кластеры (инвариант 2).
3. Написать сцену в `mockups.ts` (только ink-контент!) + добавить в объект `mockups`.
4. `npm run check` → 0 ошибок.
5. **Проверить глазами.** Скриншотер браузер-пейна в этом окружении зависает,
   поэтому рендерим офлайн: временный роут `src/pages/api/devshot.ts` (POST
   `{dataUrl, name}` → пишет PNG), `preview_start`, затем через `javascript_tool`
   импортировать реестр и нарисовать сцену на offscreen-canvas в нужных размерах
   → PNG → скопировать в `design/preview-mockups/` → посмотреть `Read`.
   Проверять: текст в границах, читаемость в размере плитки, фольга/глянец идут
   общим механизмом.
6. **Удалить** `devshot.ts` перед коммитом.
7. Выкатить по «Порядку выкатки» выше.

Референс-рендеры сцен: `design/preview-mockups/*.png` (не в гите).

## Что дальше (открытые темы)

- **Плитки каталога.** Обсуждали замену `astro-icon` на куклу. Вывод: растр
  просядет по весу/запросам, живой канвас — по INP; правильный путь —
  **инлайновый SVG, сгенерированный на сборке** (вектор, ноль запросов, ноль JS).
  Цена — адаптер «canvas-вызовы → SVG» либо отдельные SVG-функции; шум-текстуру
  для плитки отбрасываем, глянец/фольгу можно отдать градиентами.
- **Иконки кластеров.** У кластеров своё поле `promoted_pages.icon`, независимое от
  превью продукта, но сейчас у кластеров одного хаба часто стоит одна и та же
  дежурная иконка (у 3 кластеров бланков — общий `tabler:file-text`). Развести
  можно, не трогая сцены.
- **Крупноформат** — проверить `poster` вживую, когда опубликуем баннеры/плакаты.
- **Разворот вместо обложки.** Фотокнига и выпускной альбом продаются разворотом,
  а не лицом. Отдельный режим stage — кандидат, но не раньше плиток каталога.

- ✅ **Биговка перестала складываться.** `crease` (чертежи) — это продавленная
