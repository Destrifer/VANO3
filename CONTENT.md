# Наполнение контентом

Карта: **что** заполняем, **где** (поля Directus) и **статус**. Сам контент живёт
в Directus (не в репозитории) — этот файл его описывает и трекает.
Схема — в `directus/snapshot.yaml`. Обновлять статус по мере наполнения.
Последняя ревизия: 2026-07-02.

> **С 2026-06-30 контент вводится на ПРОДЕ** (`admin.printmos.ru`), не локально.
> Публикация продукта/кластера сама триггерит пересборку сайта (Flow →
> `repository_dispatch`). С 2026-07-02 правка галереи (`products_gallery`) и
> подписи файла тоже триггерит; массовая ЗАГРУЗКА файлов — нет (намеренно).

## Где живёт контент (поверхность наполнения)

| Коллекция / поле | Что | Кто заполняет |
|---|---|---|
| `products.h1` | H1 страницы (если пусто — `name`) | контент |
| `products.meta_title` / `meta_description` | SERP-сниппет (≤60 / ~150 знаков) | контент |
| `products.intro_text` | интро (вхождения головы + контекстные ссылки), 2 колонки | контент |
| `products.faq` (→ `faq_items`) | вопрос-ответ (хвост + AEO) | контент |
| `products.lead_days` | срок изготовления, рабочих дней (1/2/3) | владелец |
| `products.material_type` нет — у `papers.material_type` | тип материала для блока «Материалы» | контент |
| `products.template_file` | скачиваемый шаблон макета | контент |
| `promoted_pages` | кластерные страницы (Tier-1): slug, h1, мета, intro, `preset`, `icon`, `tile_label`, `faq` | контент |
| `papers` | материалы: `name`, `price`/`fixed_price`, `material_type`, `description` | владелец (цены) + контент (описания) |
| `finishing_options` | постпечать: `name`, `group`, цены | владелец |
| `settings.cutoff_hour` | час отсечки для «срока готовности» | владелец |
| `products.gallery` (галерея, M2M Files; junction `products_gallery`: caption/alt/sort) | примеры работ; подпись = junction.caption ИЛИ «Подпись» (title) в карточке файла | контент |

> Тексты техник постпечати (`src/lib/optionInfo.ts`) и правила (`rules.ts`) —
> осознанно в коде (см. TECHDEBT), не в Directus.

### Чеклист «довести продукт до флагмана»
1. `meta_title` / `meta_description` (с зацепкой: срок / «от» цены / гео).
2. `intro_text` — вхождения головы + 4–5 контекстных ссылок (2 колонки).
3. `faq` — 4–6 вопросов из выдачи/Wordstat (НЧ-вхождения в ответы).
4. `lead_days` — срок категории.
5. Кластеры `promoted_pages` — по дереву интентов направления (UX-дифференциация
   пресетом, не просто текст), с `icon` + `tile_label`.
6. Материалы/постпечать продукта — `material_type`, реальные цены, описания.
